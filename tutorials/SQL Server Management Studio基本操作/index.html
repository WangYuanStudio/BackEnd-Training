SQL Server Management Studio基本操作

网园资讯工作室 编程部 2017/11/05

SQL Server Management Studio 简介

百度百科解释：SQL Server Management Studio 是一个集成环境，用于访问、配置、管理和开发 SQL Server 的所有组件。SQL Server Management Studio 组合了大量图形工具和丰富的脚本编辑器，使各种技术水平的开发人员和管理员都能访问 SQL Server。

网园师兄解释：就是用来管理和操作SQL Server数据库的一个软件。


连接数据库

  1. 打开SQL Server Management Studio。


  2. 连接本地数据库。服务器类型选择数据库引擎；服务器名称默认为你电脑的计算机名，不需要更改，如果已经更改并且不知道计算机名是什么，可以填localhost；身份验证选择Windows身份验证。


  3. 如果连接失败，原因可能是因为没有开启SQL Server系统服务，这时候可以在开始菜单→所有程序里找到SQL Server配置管理器，然后左边点击SQL Server服务，右边右击SQL Server(MSSQLSERVER)，点击启动，然后再重新尝试一次连接数据库即可。


  4. 连接成功后，可在对象资源管理器里看到本地服务器。


创建数据库

  1. 在对象资源管理器中右击数据库，点击新建数据库。


  2. 在弹出的新建数据库窗口中，输入数据库名称，名称必须是英文，最好跟网站名称相同；然后将下面的滚动条拖到最右边，在路径那一列选择数据库存放的位置，为了方便交作业，请将路径设置为你的网站路径，注意两个路径要一致。


  3. 创建成功后，可在对象资源管理器中看到你创建的这个数据库。


创建数据表

数据库跟数据表的区别：一个数据库中可有多个数据表。例如，一个美食网站就用一个数据库，这个数据库中有一个数据表是存放用户信息的、有一个表是存放菜品信息的、有一个表是存放用户订单信息的、有一个表是存放餐馆财账信息的……

  1. 展开数据库左边的+号。


  2. 选中表，右击，选择新建→表。


  3. 此时会出现一个数据表的结构设计界面，根据你网站的实际情况以及你的构思来设计数据表。任何数据表都必须要有“id（下图中的uid）”这一列名，id是用于区分每一条数据的唯一标识。注意数据类型和是否允许NULL值。下图仅供参考。


  4. 数据表设计好后，选中“id”，点右键，设置主键，主键是用来唯一标识这一条数据记录的。


  5. 设置主键后，可以在“id”左侧看到一个钥匙的图标，表示这一列名是主键。设置主键后，我们还需要让“id”在记录数据时自增，这样才不会出现“id”重复的现象。在下面的列属性栏展开标识规范，将(是标识)设置为是。


  6. 完成数据表设计、主键设置、标识规范设置后，点击左上角的保存图标，或按下键盘Ctrl+S将数据表保存。


  7. 此时会弹出一个对话框让你输入表的名称，根据实际情况输入即可。表名应尽可能有意义。


  8. 点击确定后可以看到左侧对象资源管理器中已经出现了刚刚新建的表，如果没有出现，可以点击上面的刷新按钮。


表的设计及数据管理

  • 如果想对一个已经设计好的数据表进行重新设计（修改结构），可以在对象资源管理器中右击欲修改的那个数据表，选择设计。



  • 如果想查看该数据表里的数据内容，或者要手动修改数据表里的数据，可以在对象资源管理器中右击欲查看或编辑的数据表，选择编辑前200行。



数据库的分离和附加

  • 为什么要对数据库进行分离和附加？

    SQL Server 的数据库文件是以.mdf为扩展名的文件，当我们连接数据库并创建数据库后，这个文件是一直加载在数据库服务器里（即使关闭了SQL Server Management Studio）。因此要想删除mdf文件，或移动mdf文件，就必须将该文件从数据库服务器里切断，即分离。只有分离之后的独立的mdf文件，才能进行转移。从其他地方拷贝过来的mdf文件，当你想在数据库服务器里加载它时，就必须要将mdf文件附加进数据库服务器里来。

    由于我们的培训作业需要查看你们的数据库设计和结构，同时也是为了能够正常运行你们做的网站，因此你们在提交作业之前必须要将数据库分离出来，然后将mdf文件同网站一起打包后再提交作业，提交完作业后你们需要重新将mdf文件附加进你们的数据库管理器里然后继续开发你们的网站。

  • 数据库的分离。

    1. 在对象资源管理器中选中你的数据库，右击，选择任务→分离。


    1. 在弹出的窗口中点击确定即可。


    1. 分离成功后，打开你数据库文件的存放位置，将mdf文件和ldf文件一起打包提交作业即可。


  • 数据库的附加

    1. 在附加之前先修改mdf文件和ldf文件的安全属性，否则可能会附加失败。选中mdf文件（ldf文件同理），右击，选择属性。


    1. 在弹出的属性窗口中点击安全，然后选择你当前登录的系统用户，点击编辑。


    1. 将权限全部勾选允许，然后点确定关闭窗口。


    1. 接着回到数据库管理系统，在对象资源管理器中选中数据库，右击，选择附加。


    1. 在弹出的窗口中点击添加。


    1. 然后找到你数据库文件存放的位置，选择你要附加的mdf文件，点击确定。


    1. 此时你将看到你要附加的数据库列表，点击确定。


    1. 附加成功后，你可以在对象资源管理器里看到这个数据库。
